## Where We Left Off

The current frontier is no longer basic native boot or ROM/SASI handoff. Those
are working. The active blocker is deep in native `AMOSL.INI` miss handling
after the monitor has already reached:

- `QUEUEIO`
- the real file-open area around `$3720/$3748`
- `TTYLIN`
- `SCNMOD`
- `SRCH` for `AMOSL.INI`

What is proven now:

- native ROM boot runs through the normal `cpu.step()` / `bus.tick()` path
- the ROM performs real SASI-backed disk reads
- `AMOSL.MON` is loaded and reached natively
- the observed native LED sequence is `06, 0B, 00, 0E, 0F, 00`
- the native handoff milestone still reaches post-sizing `PC = 0x00811C`
- the PTM interrupt model had a real masked-IRQ bug, and that bug is fixed in
  [timer6840.py](../alphasim/devices/timer6840.py)
- the added unit coverage in
  [test_timer6840.py](../tests/devices/test_timer6840.py) passes
- native boot now reaches `SCNMOD` and `SRCH` for `AMOSL.INI`
- native `SCNMOD` calls `A06C`, and `A06C` maps directly to `SRCH` at `$1C30`
- native `SRCH` misses in memory for `AMOSL.INI` and enters the shared
  miss/setup path at `$1CFE -> A080 -> $4982`

What is still not true:

- native boot still never issues disk reads beyond LBA `3326`
- `AMOSL.INI` on this image is at LBA `3335`
- native boot still does not reach a real `TRMDEF`-driven terminal session
- `tests/integration/test_boot_native_amosl_ini.py` is still the expected
  xfail

## Most Important New Findings

### 1. PTM masked-IRQ bug was real

Before the PTM fix, the emulator would raise `_interrupt_pending` on timer
underflow even when that timer's interrupt-enable bit was clear. That caused a
spurious trip into `$0FA2` immediately after the `A060` dispatch path.

After the fix:

- the old timer-wait dead end is gone
- plain native boot reaches the real `AMOSL.INI` miss path

### 2. There is also a real low-memory ACIA RX-IRQ race

The stale `RX=$00` seen later at `$3752` comes from the low-memory poll/read
loop itself:

- `$006C3E-$006C64` generates a second echoed `NUL`
- ACIA vector 64 interrupts before `$006C52` can consume it
- the pending `NUL` survives until the later path

This race is measured, but no permanent emulator fix for it is committed.
Earlier speculative ACIA echo-model changes were reverted.

### 3. The injected `TTYLIN` harness does not prove native `AMOSL.INI` reading

`trace_scnmod.py` can feed real `AMOSL.INI` lines through COMINT/SCNMOD far
enough to process the first two `TRMDEF` lines and `VER`, but that only proves
command execution when `TTYLIN` is artificially supplied.

It does **not** prove that native `AMOSL.MON` autonomously opens and reads
`AMOSL.INI`.

### 4. Native `A06C` is not the missing promotion step

On the native `AMOSL.INI` path:

- `$3A70-$3A7C` seeds the raw descriptor in `$7074`
- `A6=$7074`
- `D6=1`
- `$A06C` is called

The LINE-A dispatcher pushes handler `$1C30`, so `A06C` is just `SRCH`. There
is no hidden live-request setup there.

### 5. Normal native miss path fails before any new disk I/O

After the memory miss:

- `$1CFE -> A080 -> $4982 -> $4A8C...$4AB4 -> $503C`

On that normal path:

- descriptor byte `+$01` ends up with bit `0x40` set
- no live request block is built
- `$503C-$5062` polls the raw descriptor at `$7074`
- byte 0 stays zero
- the path returns `D7=4`
- no reads go beyond LBA `3326`

### 6. Forced deeper branch exposes a more specific problem

If the descriptor gate is forced open so `A060` runs:

- `A060` does return a pointer
- descriptor `+$12` first becomes `$3E817C`

But then:

- `$1D12` calls `A056`
- `A056` walks the current job module chain and returns `$3E8380`
- `$1D14` immediately overwrites descriptor `+$12` with `$3E8380`

That means the original `A060` result is not merely ignored later. It is
replaced almost immediately by the module-chain-end pointer.

The later fallback family then goes:

- `$1D18 -> $1D28 -> $551C -> $5594 -> $55AA -> $5692/$56A2 -> $56BC -> $56D2`

and still launches `A086` with:

- `A4=A6=$7074`

not with a live request block.

## Current Best Diagnosis

The current best description of the native failure is:

1. `AMOSL.INI` reaches native COMINT/SCNMOD correctly.
2. Native `SRCH` looks for it in memory and misses.
3. The miss/setup path enters the shared loader machinery.
4. On the normal path, the raw descriptor is polled and fails before disk I/O.
5. On the forced deeper path, the one interesting pointer produced by `A060`
   gets overwritten at `$1D14` before the later helper family uses it.
6. Native boot therefore never turns the raw `AMOSL.INI` descriptor into a
   real disk request that would read LBA `3335+`.
