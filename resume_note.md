## Current State — TERMINAL OUTPUT WORKING

The native boot on `AMOS_1-3_Boot_OS.img` now produces real terminal
output. The OS displays the AMOS boot banner (Alpha Microsystems logo
in WYSE block graphics), "System not available" diagnostic, and the
full software license agreement — 2322+ TX bytes, all generated by
native AMOS code running on emulated hardware.

No Python bypasses, no memory patches, no LINE-A intercepts. The CPU
executes AMOS code, the ACIA hardware emulation handles interrupts,
and characters flow from the OS output buffer through the MC6850 to
the terminal.

```bash
# Run it:
python3 main.py
```

### What works natively

- ROM boot: 78 SASI reads (LBA 0-3326)
- OS SCSI driver: hundreds of reads via SCSI bus interface ($FFFFC8)
- Disk mount, AMOSL.INI loaded and processed
- COMINT processes INI: :T, JOBS 5, JOBALC, TRMDEF TRM1, TRMDEF TRM2
- AM1000.IDV loaded at $85FC, WYSE.TDV loaded at $89F4
- T.IHW = $FFFFFE20 (ACIA port 0)
- ACIA interrupt-driven TX: CHROUT enables TX → ISR calls TRMOCP →
  writes bytes to ACIA data register → next TX IRQ → repeat
- Full terminal output: escape sequences, banner, license text

### TCB field layout (AMOS 1.3 — differs from manual)

```
TCB+$00 = $0080  T.STS (status word, bit 7 = OIP)
TCB+$02 = $85FC  IDV module pointer (AM1000.IDV)
TCB+$06 = $FFFFFE20  T.IHW (ACIA port 0 hardware addr)
TCB+$0E = $89F4  TDV module pointer (WYSE.TDV)
TCB+$40 = $7038  T.JLK (JCB link)
TCB+$62 = T.INC (input char routine, 0=use TRMICP)
TCB+$66 = T.OTC (output char routine, 0=use TRMOCP)
```

### ACIA fixes that enabled output (2026-03-27)

1. **DCD=0**: AM1000.IDV ISR checks DCD before TDRE. DCD=1 skipped
   TX processing entirely (went to INFI). Real hardware has ~DCD
   tied LOW (terminal always present).

2. **TX IRQ latch**: Don't clear on RX data reads (INPR handler).
   Only clear on "dismiss" reads where RDRF was not set (INFI path
   at $0088CA).

3. **Remove RX cooldown TDRE suppression**: Echo timing workaround
   was hiding TDRE from the ISR, causing it to treat TX interrupts
   as false after processing echo bytes.

### AM1000.IDV structure (from am1000.m68 source)

```
Dispatch table:
  +$00: BR CHROUT  (output initiation — enables TX on ACIA)
  +$02: BR INIT    (port initialization — baud, reset, vector)

CHROUT: Writes CM$TXE ($B5) to ACIA control → TX IRQ fires
INIT:   Sets baud rate, resets ACIA, stores ISR vector at $068
GINTRP: Reads status → RDRF→INPR, DCD→INFI, TDRE→OUTPR
OUTPR:  Calls TRMOCP → if D1>=0: write to ACIA → if D1<0: clear OIP
INPR:   Reads data byte → calls TRMICP
INFI:   Reads data register to clear 6850 → RTE
```

### Current frontier

- Boot banner + license displayed (2322 bytes)
- Processing second TRMDEF (TRM2, port 2 at 9600 baud)
- Need >50M instructions for full INI processing
- Next commands: VER, PARITY, DEVTBL, BITMAP, SYSTEM, ATTACH

### What doesn't work yet

- Interactive terminal input (RDRF injection not tested with new ACIA)
- Commands beyond second TRMDEF (instruction limit)
- JOBTRM still $00000000 (terminal not "attached" to job yet)
- TCB+$76 (TCRT dispatch table) still zero

## Key Decoded Addresses

### Scheduler
```
$001250: TST.L ($041C).W       ; test JOBCUR
$00129C: MOVEA.L ($041C).W,A0  ; load JCB
$0012AE: MOVEA.L $80(A0),A7    ; switch to job's SSP
$0012B2: MOVEA.L $7C(A0),A6    ; load saved USP
$0012B6: MOVE A6,USP           ; set USP
$0012EE: MOVEM.L (A7)+,...     ; restore registers
$0012F2: RTE                   ; dispatch to job
```

### TRMSER output chain
```
$001FCA: LINE-A $A048 handler → checks TCB, dispatches to TDV/IDV
$001E6A: Output buffer write → $001EA8 stores byte → $001F48 checks OIP
$002AFA: TRMOCP ($A030/$A058) handler → walks buffer chain → returns char
$008844: AM1000 ISR → GINTRP → OUTPR → write to ACIA $FFFE22
```

### ACIA hardware
```
Port 0: $FFFE20 (status/control), $FFFE22 (data)
Port 1: $FFFE24/$FFFE26
Port 2: $FFFE30/$FFFE32
ISR vector: $068 (autovector level 2, vector 26)
Control $95: RX IRQ enabled, TX disabled, 8N1, div16
Control $B5: RX+TX IRQ enabled, 8N1, div16
```
